## 二、逐个产品观察

### C1. Flyability Elios 3 / Inspector

**它卖的不是炫酷 3D，而是“进入人不能进的地方，并让检查结果可信”。**

官方页面强调 Elios 3 是 indoor mapping and inspection drone，面向 inaccessible asset；核心能力包括 FlyAware SLAM、3D Live Map、3D Reporting、Return-To-Signal、抗碰撞设计、模块化 payload。页面明确说 LiDAR 会实时构建高密度点云并显示在 Cockpit App 中，让操作者获得位置感知；检查覆盖也会累计显示。

**商业面板启发**

- 第一屏应该回答：我在哪里、已经看了哪里、哪里没看、哪里危险。
- “覆盖率/已探明区域”比单纯 3D 漂亮更重要。
- 风险点需要能直接进入 3D 报告，而不是只在场景里闪一下。
- 安全边界要清楚，比如信号丢失、低置信、不可达区域、AI 推断区域。

**HIVE 差距**

- HIVE 有 3D 场景和机器人，但缺少“探索覆盖率”和“任务完成度”这一层。
- HIVE 有 AI 标记，但还不是巡检报告链条的一部分。
- HIVE 的机器人数据更像展示数据，不像一个真实任务的安全控制台。

**可融合方案**

- 新增“探明覆盖率层”：已扫描、未扫描、低置信、AI 补全四种状态。
- 风险点卡片增加“加入报告”“标记已复查”“导出证据包”。
- 任务回放从动画升级为“探索过程证据”：轨迹、覆盖率、关键发现、数据来源。

---

### C2. DroneDeploy

**它卖的是“现场真相记录”，不是单次建模。**

DroneDeploy construction 页面强调用 drones、360 cameras、robots 从各角度记录现场，形成 complete, shareable record，减少 guesswork。它把价值直接绑定到 documentation & disputes、remote visibility、schedule & planning，并明确提到 timestamped visual record、RFIs/change orders、pin issues、verify work in place、supported hardware、integrations。

**商业面板启发**

- 客户愿意为“可追责的现场记录”付费。
- 时间戳、位置、任务、证据、问题 pin，比泛泛的数字孪生更容易被买单。
- 远程团队需要共享视图，而不是每个人都打开重工具。

**HIVE 差距**

- HIVE 有截图、导出和 AI 对话，但还没有“证据链对象”。
- 告警、AI 标记、人工标注、导出报告目前是散的。
- 没有按时间组织的任务记录，也没有“这次巡检发现了什么”的商业闭环。

**可融合方案**

- 引入 `Finding` 概念：风险发现 = 位置 + 时间 + 传感器 + 点云/截图 + AI 解释 + 人工状态。
- 所有 AI 标记和告警都能转成 Finding。
- 报告不只是 PDF，而是“本次任务证据包”：PDF + CSV + 截图 + LAS/OBJ 链接 + AI 审计记录。

---

### C3. NavVis IVION

**它把现实捕获数据变成浏览器里的共享工作台，让非专家也能用。**

NavVis IVION 页面强调 high-precision reality capture data 转成 clear, collaborative digital spaces，任何角色都能 access, understand, act on。它也强调 CAD/BIM 工具往往只给专家用，而 IVION 提供 browser-based shared environment，支持 as-is 条件查看、设计模型 overlay、安装验证、问题标记、远程协作、asset tagging。

**商业面板启发**

- HIVE 应该做“非专家可理解的未知空间工作台”，不是专家建模软件。
- 浏览器体验要轻、快、能分享；专家分析交给 CAD/GIS/点云工具。
- 资产标记/问题标记/协作状态是商业化面板的关键。

**HIVE 差距**

- HIVE 的 UI 目前更像一个单机 cockpit demo，而不是协作工作台。
- 右侧详情面板展示很多参数，但缺少“这对我的工作意味着什么”。
- 没有角色视图：矿长、安全员、测绘工程师看到的信息优先级应不同。

**可融合方案**

- 新增角色视图：
  - 安全员：风险、告警、复查状态、撤离/封控建议。
  - 测绘工程师：点云质量、坐标、导出、低置信区域。
  - 管理者：任务完成度、风险趋势、报告、ROI。
- 详情面板改成“摘要优先 + 证据展开”，避免一上来砸 19 个传感器字段。

---

### C4. PIX4Dsurvey

**它不替代 CAD/GIS，而是把点云变成 CAD/GIS 更好用的成果。**

PIX4Dsurvey 官方定位是 Bridge the gap between photogrammetry and CAD。它的工作流是 Capture、Import、Vectorize、Model、Integrate；支持导入 .las/.laz，生成 TIN、CAD-ready vector files，并以 custom layers/properties 导出到 CAD/GIS。它强调“提取关键元素”，而不是把所有点都丢给用户。

**商业面板启发**

- HIVE 的导出不应只是文件按钮，而应是“导出前整理/简化/校验”的工作流。
- CAD-ready 的关键是图层、属性、坐标、单位、精简对象，而不是体积最大的数据。
- 用户愿意付费的是节省工程师清洗数据的时间。

**HIVE 差距**

- HIVE 已有 PDF/LAS/OBJ/CSV，但更像技术导出，而不是 CAD-ready 工作流。
- 没有导出前检查：坐标系、单位、图层、低置信区域、AI 推断是否包含。
- 没有把未知空间对象“矢量化/语义化”为 CAD 层。

**可融合方案**

- 导出向导：选择用途 -> 选择可信数据范围 -> 选择图层 -> 预检 -> 导出。
- 增加 CAD/GIS 图层规范：`measured_points`、`risk_findings`、`robot_path`、`low_confidence_area`、`ai_inferred_area`。
- PDF 报告明确标注：哪些是实测，哪些是推断，哪些已人工确认。

---

### C5. GroundHog Apps

**它是更重的矿山运营软件，但商业面板语言值得学：实时可见、易部署、按角色、ROI。**

GroundHog 官网覆盖 Underground FMS、RTLS、Peer-to-Peer Network、Proximity Detection、Work-order Management、Telematics、安全培训等。它强调简单、强大、综合的云平台，地下矿山像 black box，GroundHog 给实时可见性；还强调 user-friendly、configurable、affordable、部署快、培训快。

**商业面板启发**

- 矿山客户买单时关心“黑箱变透明”“更快决策”“少培训”“ROI”，不是技术栈。
- 面板应把安全、任务、人员/机器人位置、工单状态讲清楚。
- 不要一次做完整 FMS，但可以借鉴它的商业语言和关键面板。

**HIVE 差距**

- HIVE 有机器人列表，但没有真正的任务/工单/责任闭环。
- HIVE 的“管理者价值”表达不足，当前更偏技术演示。
- 没有订阅产品常见的多角色、多项目、历史记录、团队权限。

**可融合方案**

- 轻量任务闭环：发现风险 -> 指派复查 -> 状态更新 -> 关闭 -> 进入报告。
- 管理者首页：今日风险、未复查项、覆盖率、机器人在线率、导出报告。
- 订阅价值表达：减少危险进入、缩短巡检报告、降低专家判读成本。

## 三、HIVE 与同类产品的客观差距

### 总体判断

“现在像游戏/玩具”这个感受是成立的，但要说清楚：HIVE 的概念方向并不玩具，当前玩具感主要来自**商业闭环、真实数据链、证据链、任务流、可信边界**还没有完全长出来。

HIVE 目前像一个能力很丰富的可视化 demo：有 3D、有机器人、有 AI、有导出、有场景切换。但商业产品卖的不是“我能显示这些东西”，而是“我帮你在危险未知环境里更快做出可信决策，并留下可交付证据”。

### 差距总表

| 维度 | HIVE 现在 | 同类商业产品 | 差距判断 | 优先级 |
|------|-----------|--------------|----------|--------|
| 真实采集链路 | 以 mock 为主，live 后端可选 | 绑定无人机/相机/激光雷达/机器人真实采集 | 最大差距之一 | P0 |
| 任务闭环 | 有场景和标注，但任务状态弱 | 巡检、覆盖、报告、复查闭环明确 | 明显差距 | P0 |
| 证据链 | 有数据和导出，但没有 Finding 对象 | 时间戳、位置、影像/点云、问题 pin、报告 | 明显差距 | P0 |
| 可信边界 | 有物理真实模式，但表达偏粗 | 区分 capture、processing、inference、human verified | 明显差距 | P0 |
| 商业面板 | 信息密集，偏工程 demo | KPI、任务、风险、ROI、角色视图清晰 | 明显差距 | P1 |
| AI 交互 | 有自然语言和场景动作 | 同类多数 AI 仍浅，HIVE 有机会领先 | 机会点 | P0 |
| 导出 | 已有 PDF/LAS/OBJ/CSV | CAD-ready、图层、预检、集成 | 有基础但不产品化 | P1 |
| 协作 | 单人 cockpit | 多人共享、评论、分派、历史 | 中等差距 | P2 |
| 视觉成熟度 | 有酷炫 3D，但缺少商业控件质感 | 产品面板更克制、更任务导向 | 中等差距 | P1 |

## 四、分别对标融合路线

### 对标 Flyability：从“3D 场景”升级成“探索覆盖面板”

要补：
- 探索覆盖率层：已探明、未探明、低置信、AI 推断。
- 机器人轨迹与信号状态。
- 巡检完成度和遗漏区域。
- 风险点一键进入报告。

做完后，HIVE 会从“好看的地下场景”变成“我知道这次任务看到了哪里、没看到哪里、哪里危险”的工具。

### 对标 DroneDeploy：从“告警/标注”升级成“证据链”

要补：
- Finding 对象。
- 每个 Finding 有状态：新发现、已确认、已指派、已复查、已关闭。
- 每个 Finding 绑定：位置、时间、机器人、传感器、截图、点云片段、AI 解释。
- 报告按 Finding 生成，而不是按页面截图拼装。

做完后，HIVE 可以卖“减少争议、减少漏检、缩短报告时间”。

### 对标 NavVis IVION：从“专家控制台”升级成“非专家也能理解的空间工作台”

要补：
- 角色视图。
- 摘要优先，参数展开。
- 共享链接/只读报告视图。
- 资产/区域/问题标签。

做完后，矿长、安全员、测绘工程师不用都面对同一套复杂面板。

### 对标 PIX4Dsurvey：从“导出按钮”升级成“CAD-ready 交付”

要补：
- 导出向导。
- 图层与属性标准。
- 导出前校验。
- 低置信/AI 推断是否包含的显式开关。
- 简化/矢量化对象：风险点、机器人轨迹、边界、覆盖区、通道中心线。

做完后，HIVE 不需要替代 CAD，但能成为 CAD 前最省事的数据整理工具。

### 对标 GroundHog：从“技术 demo”升级成“矿山商业面板”

要补：
- 管理者首页。
- 任务/工单轻闭环。
- 安全 KPI：未复查高危项、机器人在线率、覆盖率、近 24h 趋势。
- 部署/订阅价值表达：少培训、快上线、低成本、可导出。

做完后，HIVE 才更像可卖软件，而不是展示项目。

## 五、推荐产品路线

### 第一阶段：去玩具感，建立可信商业闭环

目标：让用户相信 HIVE 不是“3D 游戏”，而是安全决策工具。

交付：
- Finding 风险发现对象。
- 证据卡片。
- 探索覆盖率。
- 实测/推断/AI 补全分层。
- 报告从 Finding 生成。

### 第二阶段：AI 成为核心交互，而不是聊天外挂

目标：AI 负责帮用户查、问、筛、解释、生成任务，但必须有边界。

交付：
- AI 动作权限表。
- AI 回答引用证据。
- AI 不能确认安全，只能给“建议/风险等级/需要人工复核”。
- 所有 AI 动作可撤销、可审计。

### 第三阶段：导出成为付费壁垒

目标：把 HIVE 变成重工具前的数据整理入口。

交付：
- CAD/GIS 导出向导。
- 图层模板。
- 坐标/单位/置信度预检。
- 报告历史。
- 批量导出。

### 第四阶段：商业订阅面板

目标：让不同角色每天打开它。

交付：
- 管理者首页。
- 安全员复查队列。
- 测绘工程师数据质量页。
- 项目历史和团队权限。

---

## 六、明确不建议现在做的事

- 不做完整 CAD 编辑器。
- 不做完整矿山 FMS。
- 不做重资产管理平台。
- 不做复杂 ERP/CMMS。
- 不做泛泛“数字孪生中台”叙事。

这些可以导出或集成，但不是 HIVE 当前最该自己吞下的范围。

## 八、深研版对比框架（2026-06-13）

### 8.1 这次对标的前提

HIVE 与这些国际产品的载体不完全一样：

- Flyability / DroneDeploy / NavVis 很多价值来自相机、全景影像、无人机/360 相机/机器人采集。
- HIVE 的特种机型更偏传感器、点云、轨迹、环境指标和未知空间推断，不应硬抄视频/影像工作流。
- 但终端用户打开面板时的诉求相似：快速知道哪里危险、哪里已经查过、证据是否可信、下一步该做什么、能否导出给专家/监管/客户。

因此对标重点不是“摄像头功能”，而是：

1. 如何把未知空间变成可理解状态。
2. 如何让非专家也能判断风险。
3. 如何证明数据可信。
4. 如何把发现沉淀成报告/任务/导出物。
5. 如何让客户愿意付费。

### 8.2 评分维度

每个维度 0-10 分，满分 100。评分不是公司规模评分，而是“对 HIVE 当前商业化方向的参考价值与成熟度”。

| 维度 | 含义 |
|------|------|
| 行业聚焦 | 是否清楚服务什么场景、什么角色、什么购买理由 |
| 采集链路 | 是否从硬件/载体/数据到软件面板形成闭环 |
| 未知空间认知 | 是否清楚表达已探明/未探明/风险/位置/覆盖 |
| 面板易用性 | 非专家是否能看懂和行动 |
| 证据链 | 是否记录时间、位置、来源、发现、复核、报告 |
| 安全与可信边界 | 是否表达数据置信、限制、风险边界 |
| 报告/交付 | 是否能形成客户可交付成果 |
| 导出/集成 | 是否能进入 CAD/GIS/BIM/工单/企业系统 |
| 商业闭环 | 是否有清楚付费理由、ROI、角色价值 |
| AI/自动化潜力 | 是否有可被 AI 放大的交互空间 |

## 九、逐产品深研与 HIVE 差距

### 9.1 Flyability Elios 3 / Inspector

**服务行业**

- 密闭空间、矿山、油气、化工、发电、船舶、污水、储罐、烟囱、管廊等危险/不可达检查。
- 核心用户是巡检人员、安全负责人、资产 owner、检测服务商。

**核心能力**

- 抗碰撞室内无人机进入危险空间。
- FlyAware SLAM 定位与 3D Live Map。
- 实时点云/空间定位，帮助操作者知道无人机在哪里。
- 检查覆盖显示，降低漏检。
- Inspector 软件用于检查数据复盘、缺陷定位、3D Reporting。
- 可扩展 payload，例如 LiDAR、UT 厚度测量、辐射探测等。
- Return-To-Signal 等安全能力，强调任务安全边界。

**它最后怎么展示**

- 操作者侧：飞行界面 + 实时 3D 地图 + 位置感知 + 覆盖状态。
- 复盘侧：3D 模型/点云中的 POI、检查照片/测量、报告输出。
- 商业表达不是“我们有 3D”，而是“你不用人进去，也能安全完成检查并交付报告”。

| 项 | Flyability | HIVE 当前 |
|----|------------|-----------|
| 行业聚焦 | 9 | 6 |
| 采集链路 | 9 | 3 |
| 未知空间认知 | 9 | 5 |
| 面板易用性 | 8 | 5 |
| 证据链 | 8 | 3 |
| 安全与可信边界 | 8 | 4 |
| 报告/交付 | 8 | 5 |
| 导出/集成 | 7 | 5 |
| 商业闭环 | 9 | 4 |

## 十一、补强清单：采众家之长但不超出 HIVE 形态

### P0：必须补，否则仍像 demo

1. **Finding 风险发现对象**
   - 来源：DroneDeploy issue pin + Flyability inspection POI。
   - HIVE 版本：风险点 = 位置 + 传感器 + 置信度 + 轨迹 + AI 解释 + 复查状态。

2. **探索覆盖率**
   - 来源：Flyability coverage / 3D Live Map。
   - HIVE 版本：不用视频，用机器人轨迹、传感器采样点、点云密度判断已探明区域。

3. **可信边界分层**
   - 来源：安全软件共同原则。
   - HIVE 版本：实测 / 插值 / AI 推断 / 未探明 四层常驻可见。

4. **AI 证据引用**
   - 来源：HIVE 自身 AI 优势。
   - HIVE 版本：AI 每个结论必须引用传感器、位置、时间、置信度；没有证据就说“不足以判断”。

5. **任务报告从 Finding 生成**
   - 来源：Flyability report + DroneDeploy record。
   - HIVE 版本：报告不是截图集合，而是发现清单、证据链、风险等级、导出附件。

### P1：做完才像可卖产品

1. **角色首页**
   - 管理者：风险总览、未复查、覆盖率、报告。
   - 安全员：高危队列、处理状态、复查任务。
   - 工程师：点云质量、低置信区、导出。

2. **导出向导**
   - 来源：PIX4Dsurvey。
   - HIVE 版本：用途选择、图层选择、可信边界、预检、导出历史。

3. **轻量任务闭环**
   - 来源：GroundHog 工单/FMS，但只取最小闭环。
   - HIVE 版本：风险发现 -> 指派复查 -> 上传/录入结果 -> 关闭。

4. **商业指标**
   - 减少人工进入次数。
   - 缩短报告时间。
   - 高危发现响应时间。
   - 未探明区域比例。

### P2：增强订阅价值

1. 多项目历史。
2. 团队权限。
3. 共享只读链接。
4. 审计日志。
5. 报告模板。
6. CAD/GIS 图层模板库。
