# AI Prompt 通用规则标准

> **文档级别**：强制规范（Mandatory）
>
> **代码真源**：`src/api/aiApi.ts` — `COMMON_*` 常量 + `SCENE_PROMPTS` 配置表
>
> **核心原则**：通用规则集中维护，场景只维护特有部分，新增场景只需写一个配置项。

---

## 一、架构：通用规则 + 场景配置

最终发给 LLM 的 system prompt 由 **3 层** 自动拼装：

```
┌─────────────────────────────────────────────┐
│ Layer 1: 通用能力（COMMON_ABILITY）           │  ← 所有场景共用
│   言出法随概念 + 工具调用约束                  │
├─────────────────────────────────────────────┤
│ Layer 2: 场景特有（SCENE_PROMPTS[scenario]）  │  ← 每场景独立
│   身份定位 + 行业知识 + 工具示例 + 专属规则     │
├─────────────────────────────────────────────┤
│ Layer 3: 通用规范（COMMON_RULES）             │  ← 所有场景共用
│   语言/格式/分级 + 防幻觉约束                  │
└─────────────────────────────────────────────┘
```

**新增场景时**：只需在 `SCENE_PROMPTS` 中加一个配置项，Layer 1 和 Layer 3 自动注入。

## 二、通用规则内容（Layer 1 + Layer 3）

### COMMON_ABILITY — 言出法随

```
你可以通过调用工具直接操控3D场景。当用户要求查看某个位置、标记危险点、测量、切换场景时，
务必调用对应工具而不是仅仅文字描述。

重要：能用工具操作的就不要只用文字回答！用户要的是看到场景变化。
```

### COMMON_RULES — 回答规范

```
- 使用中文回答
- 引用具体数值时必须标注单位
- 给出分级的建议（正常/关注/警告/危险）
- 回答要简洁专业，避免空泛
```

### ANTI_HALLUCINATION — 防幻觉约束（7 条硬规则）

| # | 规则 | 说明 |
|---|------|------|
| 1 | 只用实时数据 | 绝不编造任何未提供的具体数值 |
| 2 | 数据不足要说明 | 必须说"当前数据不足以做此判断"，不猜测 |
| 3 | 不推测未接入参数 | 说"该参数当前未接入，建议查看其他指标" |
| 4 | 标准号要准确 | 不确定编号时说"参考相关行业安全标准"而非编造 |
| 5 | 区分实测与推断 | 推断时必须加"推测"前缀 |
| 6 | 安全建议保守 | 宁可多报风险，不可漏报。不确定时建议"暂停作业并请专家到场" |
| 7 | 不编造 ID | 只能引用系统提供的清单中的真实裂缝/管道 ID |

## 三、场景配置项（Layer 2）

每个场景配置 5 个字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| `name` | AI 助手名称 | `"GeoAssist"` |
| `identity` | 专家身份 + 服务对象 | `煤矿地质安全工程师，服务矿长` |
| `toolExamples` | 场景特有的工具调用示例 | `找危险点→mark+fly` |
| `extraRules` | 场景特有的回答规则（标准号等） | `引用 AQ 1029` |
| `knowledge` | 行业领域知识 | `CH₄ 阈值、渗透率范围...` |

## 四、新增场景 Checklist

新增一个场景只需：

1. **在 `src/types/index.ts`** 的 `ScenarioType` 联合类型中添加新场景 key
2. **在 `src/api/aiApi.ts`** 的 `SCENE_PROMPTS` 中添加一个配置项（5 个字段）
3. **在 `src/api/llmTools.ts`** 的 `buildSceneContext` 中补齐该场景的 Record map 条目
4. **不需要** 重复写防幻觉约束、言出法随说明、回答格式规范 — 它们自动注入

---

## 五、PR Review 检查项

| 检查项 | 方法 |
|--------|------|
| 新场景是否配置了 `SCENE_PROMPTS` | grep 新场景 key in `aiApi.ts` |
| `buildSceneContext` 所有 Record map 是否包含新场景 | 检查 7 个 Record map |
| 防幻觉约束是否被注入 | `getSystemPrompt` 返回值末尾必须含 `ANTI_HALLUCINATION` |
| 通用规则是否被注入 | 返回值必须含 `COMMON_ABILITY` 和 `COMMON_RULES` |
