HIVE UI/UX 国际化工业质感改造方向

目标不是把产品做成更炫，而是让现场安全员、工程师、管理者一眼知道任务状态、风险证据、复查边界和交付能力。下面三版都保留现有“未知空间安全认知工作台”的产品定位，但在色调、信息层级和商业化气质上拉开差异。

三版色调定位

版本主色调气质适合
A石墨黑 / 冷灰 / 安全绿 / 克制琥珀最稳、最国际工业软件Flyability / NavVis 风格
B深海蓝黑 / 青蓝 / 电光绿更像指挥中心，更有科技感DroneDeploy / GroundHog 指挥屏
C浅灰 SaaS 外壳 / 深色 3D 画布最易用、最像订阅制 Web 产品非专家、管理者、国际客户演示

详细改造表

模块当前问题改造方式产品收益
颜色系统黄黑高饱和面积过大，容易像游戏 HUD黄色降级为选中/主按钮；背景用中性灰黑或深蓝；风险色只服务状态更可信、更耐看，减少视觉疲劳
字体Oswald 全局使用，中文正文偏紧张正文改 Inter / Noto Sans SC；数字用 JetBrains Mono；品牌少量保留窄体国际化阅读体验明显提升
顶部栏工具、场景、时间、导出并列，层级不够左品牌，中任务/场景，右角色/语言/导出；弱化技术标签用户先理解“我在哪个任务里”
左侧栏告警、机器人、系统状态同时抢注意力先放 Mission Snapshot / Risk Queue / Fleet Health；原始参数下沉管理者和安全员更快行动
右侧栏参数很多但缺少复查优先级详情页先给结论卡：状态、可信边界、证据、建议动作，再给参数表从“看数据”变成“做决策”
3D 画布悬浮标签太多，调试感强默认只显示高风险、当前选中和任务路径；其它按层/悬停显示场景更干净，空间关系更清楚
AI 面板像聊天工具，不像受控安全助手加入 evidence chips、动作审计、复查提示、可撤销状态AI 从“聊天”变成可审计工作流
导出中心功能强，但商业交付感可再加强改为 Export Package：用途、边界、预检、历史、重工具衔接更像付费壁垒
空/错状态现场失败时用户不知道下一步每个错误给原因、影响、下一步动作、是否可跳过降低现场使用焦虑
